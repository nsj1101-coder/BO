
# CMS 기반 단계별 CRM 확장 기획서
> 목적: 현재 랜딩페이지 중심 CMS 위에 단계별 CRM(리드 CRM → 세일즈 CRM → 마케팅 CRM → 행동 기반 CRM)을 확장하기 위한 상세 IA, 기능 정의, UI 구조, 사용자 플로우, 데이터 구조, 리스트/상세 기준을 정리한다.  
> 사용 대상: Claude, 기획자, 디자이너, 프론트엔드/백엔드 개발자  
> 문서 성격: 메뉴 구조도 + 기능명세 초안 + 화면 설계 기준 + 데이터 모델링 기준

---

# 1. 프로젝트 전제

## 1.1 현재 상태
현재 시스템은 **랜딩페이지 중심 CMS**이며, 아래 기능이 이미 구현된 상태로 간주한다.

- 메인 페이지 관리
- 서브 페이지 관리
- 템플릿 관리
- 템플릿 코드 편집 및 버전관리
- 팝업 관리
- 게시판/FAQ 관리
- 상담 신청 관리
- 문의 관리
- 예약 관리
- 관리자 계정/권한 관리
- 회원가입/회원 관리
- 유저 페이지 렌더링

즉, 현재는 **콘텐츠 운영형 CMS**는 성립되어 있으며,  
고객 접점 데이터(문의/상담/예약/회원가입)가 이미 존재하므로  
**리드 수집형 CRM**을 붙일 수 있는 기본 조건은 충족되어 있다.

## 1.2 CRM 도입 방향
현재 구조에서는 처음부터 풀스펙 CRM을 만들기보다 아래 4단계로 확장하는 것이 현실적이다.

### 1단계. 리드 CRM
- 문의/상담/예약 통합
- 고객 카드 생성
- 유입 경로 저장
- 상태값 관리
- 담당자 지정
- 상담 메모
- 후속 일정
- 계약 여부 체크

### 2단계. 세일즈 CRM
- 견적
- 계약
- 파이프라인
- Won / Lost
- 실패 사유
- 예상 매출

### 3단계. 마케팅 CRM
- UTM
- 캠페인
- 랜딩별 전환율
- 매체 성과 분석
- 문자/메일 발송 이력

### 4단계. 행동 기반 CRM
- 로그인 서비스
- 주문/결제
- 기능 사용 로그
- 마이페이지
- 서비스 이용 이력

# 3. 단계별 CRM 상세 정의

---

# 3-1. 1단계 리드 CRM

## 3-1-1. 도입 목적
랜딩페이지에서 유입되는 고객 접점을 하나의 고객 단위로 통합하여,  
상담/문의/예약/회원가입 데이터를 한 사람 또는 한 업체 기준으로 관리할 수 있게 한다.

## 3-1-2. 핵심 메뉴 구조

```text
CRM
└─ 리드 CRM
   ├─ 리드 통합함
   ├─ 고객 카드
   ├─ 상담 관리
   ├─ 후속 일정
   ├─ 담당자 관리
   └─ 상태 코드 관리
```

---

## 3-1-3. 메뉴별 상세 기능 정의

### A. 리드 통합함

#### 목적
문의/상담/예약/회원가입 등 서로 다른 접점 데이터를 한 화면에서 조회하고,  
중복 여부를 확인하여 고객 카드로 전환하는 허브 화면.

#### 주요 기능
- 전체 리드 목록 조회
- 리드 유형 필터
  - 문의
  - 상담
  - 예약
  - 회원가입
  - 수동 등록
- 상태 필터
  - 신규
  - 확인중
  - 접촉완료
  - 후속대기
  - 계약검토
  - 계약완료
  - 종료
- 담당자 필터
- 기간 필터
- 유입채널 필터
- 검색
  - 이름
  - 연락처
  - 이메일
  - 회사명
- 중복 리드 탐지
- 고객 카드 생성
- 기존 고객 카드에 연결
- 메모 작성
- 우선순위 지정
- 리드 상세 팝업/사이드패널
- 상태 변경
- 담당자 지정
- 후속 일정 생성

#### 리스트 컬럼
- 체크박스
- 리드 ID
- 접수일시
- 리드 유형
- 고객명
- 연락처
- 이메일
- 회사명
- 유입 채널
- 랜딩 페이지
- 상태
- 담당자
- 최근 접점일
- 다음 액션일
- 중복 여부
- 중요도
- 액션 버튼

#### 상세 항목
- 기본 정보
- 제출 원본 데이터
- 리드 생성 경로
- UTM/리퍼러
- 메모
- 상태 변경 이력
- 담당자 변경 이력
- 관련 일정
- 관련 고객 카드 연결 정보

#### UI 구조
- 상단: 제목 + 총 건수 + 신규 건수 + 오늘 유입 건수
- 필터 바: 리드유형/상태/담당자/채널/기간/검색
- 본문 좌측: 리스트 테이블
- 본문 우측: 선택 리드 상세 패널
- 상세 패널 탭:
  - 기본 정보
  - 제출 데이터
  - 이력
  - 메모
  - 액션

#### 사용자 플로우
1. 운영자가 리드 통합함 진입
2. 신규 리드만 필터링
3. 특정 리드를 선택
4. 중복 고객 여부 확인
5. 신규 고객이면 고객 카드 생성
6. 기존 고객이면 기존 카드 연결
7. 담당자 지정
8. 상태를 "확인중" 또는 "접촉완료"로 변경
9. 후속 일정 등록
10. 상담 관리 또는 세일즈 단계로 이동

### B. 고객 카드

#### 목적
고객 1명 또는 업체 1개를 중심으로, 모든 접점 이력과 상태를 통합 관리하는 기준 데이터.

#### 주요 기능
- 고객 카드 생성/수정
- 회원/비회원 통합 관리
- 개인/법인 구분
- 고객 태그 부여
- 고객 등급 설정
- 수신동의 관리
- 블랙리스트 설정
- 중복 고객 병합
- 타임라인 조회
- 관련 리드/상담/예약/계약 연결
- 고객 메모
- 첨부파일 관리

#### 리스트 컬럼
- 고객 ID
- 고객 구분
- 이름/업체명
- 연락처
- 이메일
- 담당자
- 현재 상태
- 등급
- 최근 접점일
- 보유 리드 수
- 계약 여부
- 예상 매출
- 태그

#### 상세 화면 섹션
- 기본 정보 카드
- 상태/담당자/등급 카드
- 접점 타임라인
- 관련 리드 목록
- 관련 상담 목록
- 관련 예약 목록
- 관련 견적/계약 목록
- 메모
- 첨부 파일
- 마케팅 수신 동의
- 유입 정보 요약

#### UI 구조
- 상단: 고객명/업체명 + 상태 배지 + 담당자 + 주요 액션 버튼
- 본문 2단 분할
  - 좌측: 기본정보, 태그, 연락처, 동의여부
  - 우측: 타임라인, 메모, 관련 데이터
- 하단 탭
  - 리드
  - 상담
  - 예약
  - 견적
  - 계약
  - 발송이력
  - 행동로그

#### 사용자 플로우
1. 운영자가 고객 카드 진입
2. 기본정보 확인
3. 타임라인으로 최근 접점 확인
4. 메모 또는 활동기록 추가
5. 필요 시 상태 변경
6. 후속 일정 등록
7. 영업 파이프라인 또는 계약 단계로 이동

---

### C. 상담 관리

#### 목적
고객과의 실제 상담 기록을 표준화하여 축적하고, 상담 결과가 후속 액션 및 세일즈로 연결되도록 한다.

#### 주요 기능
- 상담 등록/수정
- 상담 유형 관리
  - 전화
  - 방문
  - 카카오톡
  - 이메일
  - 화상회의
- 상담 결과 기록
- 상담 요약 기록
- 상담 성공/보류/실패 처리
- 재연락일 설정
- 고객 카드 연결
- 담당자 지정
- 첨부파일 업로드
- 상담 스크립트/템플릿 참조

#### 리스트 컬럼
- 상담 ID
- 상담일시
- 고객명
- 상담 유형
- 담당자
- 결과
- 다음 액션일
- 관련 리드
- 요약

#### 상세 항목
- 상담 기본정보
- 고객 정보
- 상담 내용
- 주요 니즈
- 예산
- 희망 일정
- 요청 서비스
- 상담 결과
- 실패 사유
- 다음 액션
- 첨부파일

#### UI 구조
- 상단: 상담 기본 카드
- 본문:
  - 좌측: 고객/리드 정보
  - 우측: 상담 기록 입력 폼
- 하단: 관련 메모 및 이력

#### 사용자 플로우
1. 리드 또는 고객 카드에서 상담 생성
2. 상담 유형 선택
3. 내용 기록
4. 결과 저장
5. 후속 일정 생성 또는 세일즈 파이프라인 이동

---

### D. 후속 일정

#### 목적
누락 없는 고객 후속조치를 위해 일정과 할 일을 고객/리드 중심으로 관리한다.

#### 주요 기능
- 일정 등록/수정/완료 처리
- 할 일 구분
  - 전화 예정
  - 견적 발송
  - 재상담
  - 자료 요청
  - 계약 확인
- 고객 연결
- 리드 연결
- 담당자 지정
- 마감일 설정
- 알림 설정
- overdue 표시
- 캘린더 보기 / 리스트 보기

#### 리스트 컬럼
- 일정 ID
- 제목
- 고객명
- 담당자
- 상태
- 예정일
- 우선순위
- 관련 단계
- 생성일

#### UI 구조
- 상단 탭: 리스트 / 캘린더
- 좌측 필터: 담당자/상태/기간/우선순위
- 본문 리스트
- 우측 상세 패널

#### 사용자 플로우
1. 리드 처리 중 후속 일정 생성
2. 담당자에게 자동 귀속
3. 예정일 도래
4. 완료 또는 연기 처리
5. 결과 메모 기록

---

### E. 담당자 관리

#### 목적
CRM 내 고객/리드/상담/일정 데이터를 담당자 중심으로 분배하고 처리 성과를 추적한다.

#### 주요 기능
- 담당자 목록 조회
- 담당 리드 수
- 담당 고객 수
- 진행중 상담 수
- 금주 처리 건수
- 자동 배정 규칙
- 수동 재배정
- 부재중 대체 담당자 설정

#### UI 구조
- 카드형 담당자 현황
- 담당자별 KPI
- 배정 규칙 설정 모달

---

### F. 상태 코드 관리

#### 기본 상태 예시
- 신규
- 확인중
- 접촉완료
- 후속대기
- 제안준비
- 계약검토
- 계약완료
- 종료

#### 주요 기능
- 상태 코드 추가/수정/삭제
- 순서 변경
- 색상 지정
- 기본값 설정
- 다음 단계 정의
- 종료 상태 정의

# 8. 데이터 연계 원칙

## 8.1 고객 중심 구조
모든 데이터는 최종적으로 `customer_id` 기준으로 묶이는 것을 원칙으로 한다.

연결 방향:
- 문의 → lead
- 상담 → consultation
- 예약 → lead 또는 reservation
- 회원가입 → member + customer
- 견적 → quotation → customer
- 계약 → contract → customer
- 행동로그 → user_behavior_event → customer/member

## 8.2 원본 보존 원칙
폼으로 들어온 원본 값은 별도 raw 데이터로 보존한다.
- 원본 폼 데이터
- 가공된 CRM 데이터
- 상태 변경 이력
- 담당자 변경 이력
- 병합 이력

## 8.3 이력 중심 구조
CRM은 수정본만 남기면 안 되고, 항상 이력이 남아야 한다.
- lead_status_history
- assignee_history
- consultation_history
- sales_stage_history
- contract_history
- message_send_history

# 10. 핵심 요약

## 10.1 지금 구조에서 가능한 것
현재 CMS는 랜딩 중심이지만,  
이미 문의/상담/예약/회원가입 접점이 있으므로 **리드 CRM은 즉시 가능**하다.

## 10.2 지금 구조에서 바로 해야 할 것
1. 문의/상담/예약 데이터 통합
2. 고객 카드 생성
3. 담당자 배정
4. 상태값 관리
5. 후속 일정 관리
6. 상담 기록 표준화

## 10.3 이후 확장
리드 CRM 위에 세일즈 CRM, 마케팅 CRM, 행동 기반 CRM을 단계적으로 확장하면  
단순 CMS를 넘어 **운영 + 영업 + 마케팅 + 고객 행동 분석 플랫폼**으로 확장 가능하다.
