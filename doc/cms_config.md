4.2 관리자는 메인 페이지 관리에 접속해서, 모든 영역들을 컨테니어(템플릿)별로 소스코드를 수기로 입력해서 메인 페이지를 관리 할 수 있어야함 
4.3 소스코드는 tailwind,html,css,js가 포함 되어 있음
4.4 대부분의 디자인 파일은 pencil ai에서 추출한 .pen 파일 바탕으로 할것임
4.5 나중에는 .pen 파일을 업로드하면 claude가 자동 인식해서 해당 영역들을 컨테이너별로 나눠서 템플릿을 등록 해줘야함 
4.6 템플릿==컨테이너, 즉 컨테이너별로 템플릿이 등록 되어야함 
4.7 템플릿에는 그러면 header 템플릿, footer 템플릿, 이렇게 모든 영역들을 컴포넌트화 시켜서 CRUD가 가능해야함

5. 서브 페이지는 무슨 페이지인가? 
5.1 page?id=1, page?id=2.... 이렇게 서브 페이지들을 생성 할 수 있어야함 
5.2 서브 페이지들은 여러 페이지일 수 있으며 등록과 동시에 메인의 상/하단 고정 되어 있는 템플릿이 기본으로 깔려야함, 그러고 가운데에 비어 있는 영역들은 서브 페이지마다 또 템플릿을 추가해서 퍼블리싱 할 수 있어야함 

6. 템플리 관리는 무엇인가? 
6.1 컨테이너별로 메인 페이지를 퍼블리싱 햇지만, 차후에 다른 프로젝트를 퍼블리싱할때 쉽게 가져와서 쓸 수 있게 만들어놓은 기능, 서브 페이지에도 메인이랑 중복되는 영역이 있을수도 있으니, 유용하게 컴포넌트 처럼 쓰려고 만들어놓은 기획 
6.2 템플릿에 "A" 템플릿을 등록하면 템플릿 A는 고유한 소스임, 다만 메인 페이지나 서브페이지에서 "템플릿 A"를 등록하면, 해당 템플릿은 "템플릿A_1"이 되어야함, 왜냐하면 안에 텍스트나 이미지나 스타일을 살짝씩 바꿀 수 있어야하는데, 템플릿 관리에서 등록 된 "템플릿A"는 고유한 소스로 관리 되어야하고,
메인페이지나 서브페이지에서는 템플릿A_1,템플릿A_2와 같이 별도로 관리 되어야함, 쉽게 얘기하면 템플릿 관리에서 컨테이너 소스를 등록하고 메인 페이지나 서브 페이지 관리에서는 해당 소스를 복사한다는 느낌으로 되면 좋을듯 
또한 메인페이지의 템플릿A와 서브페이지의 템플릿A들은 등록되는 이미지가 다를수도 있으니, 이 이미지들을 관리 할 수 있는 기능들도 필요함 
6.3 각 템플릿마다 배너 치환코드를 넣어서, 배너를 유동적으로 관리 할 수 있으면 좋을듯, 예를 들면 {{BANNER:1}}이라는 코드를 템플릿 안에 넣고, 템플릿 리스트나 상세에서 한개의 배너를 등록하면 {{BANNER:1}}에서 배너가 노출 되면 좋을듯

1. 게시판 관리 메뉴 만들기 
 1.1. 게시판은 리스트형,앨범형 유형이 있어야함, 해당 유형을 선택해서 게시판을 만들 수 있어야합니다. 리스트형과 앨범형의 UI는 알아서 디자인, 최신 게시판 형태로, 상/하단은 메인 페이티 관리에 있는 템플릿 들중, 상단, 하단 고정 된 것이 붙여질 것임(너가 붙이면 안 되고 사이트 설정에 따라가야한다는 뜻임)
 1.2. 게시판은 서브 페이지와 동일하게 여러 게시판이 유동적으로 만들어져야합니다.

 1.4 게시판 상세(게시글 상세지)는 일반적인 게시판 상세로 알아서 너가 UI를 잡아줘야함, 앞선 설정들에 따른 항목들, 뭐 제목,날짜,작성자,글내용,댓글 등등,, 전부 다 노출 시켜줘야함, 게시판 상세(게시글 상세)또는 메인 페이지의 템플릿 들중 상/하단 고정 된 것들이 자동으로 뜰거임
